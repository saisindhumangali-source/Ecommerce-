1. Abstract

In the modern era of digital retail, transitioning small and medium sellers from offline models to online platforms is essential for rampant growth. Therefore, as software engineers, our job is to ease the path of this transition for the seller. Amongst many things that an online site requires, the most important is a highly secure and optimized database system.

This project addresses this need by designing and developing a robust E-Commerce Database Management System (DBMS). The primary objective is to engineer a highly normalized relational database capable of securely managing complex operations such as inventory tracking, user authentication, shopping carts, and dynamic financial transactions.

2. Introduction

The Prime Objective of our database project is to design a robust E-commerce database by performing critical backend operations such as viewing orders, placing orders, updating the database in real-time, reviewing products, and maintaining strict data consistency across tables.

2.1 Project Requirements

  • A Customer can see account details and update them if required.
  • Customer can search for products according to the category.
  • Customer can add wishlist items to the cart and see the total amount.
  • Customer can update the cart whenever required.
  • Customer can choose the mode of payment.
  • Customer can keep track of the order by seeing the order status.
  • Customer can review products which have been purchased.
  • Seller can update the stock of a particular product dynamically.
  • Seller can keep track of total sales and filter by day, month, or year.

2.2 Functional Requirements

  • A Customer cannot access the Seller details and vice-versa (Role-based access).
  • There should not be any inconsistency in the data.
  • There should not be any loss of data during transactions.

7. Backend Implementation & Queries

To ensure absolute data integrity and automate business logic at the database level, advanced SQL features were implemented:

7.1 Stored Procedures

  • Customer Details: Procedure to securely fetch details of a customer.
  • Order History: Procedure for retrieving a user's entire order history.

7.2 SQL Triggers (Automated Inventory Management)

  • Stock Management: A trigger is designed to update the number of products as soon as a payment is made successfully.
  • Cart Calculation: A trigger to automatically update the total amount for a user every time they add something to the payment/cart table.

7.3 Complex Analytics Queries

The database is capable of fulfilling complex analytical queries required by sellers and administrators, such as:

  • Query 1: Finding products with the highest ratings for a given category.
  • Query 2: Filtering out products according to their brand and price.
  • Query 4: Finding the best seller of a particular product.
  • Query 6: Listing the product whose sale is the highest on a particular day.
  • Query 8: Listing the customers who bought products from a particular seller the most.
  • Query 9: Listing all orders whose payment mode is not CoD and are yet to be delivered.
  • Query 10: Listing all orders of customers whose total amount is greater than 5000.
  • Query 12: Listing the seller who has the highest stock of a particular product.

8. Frontend UI Implementation (React.js)

While the database acts as the source of truth, a decoupled React.js user interface was engineered to prototype the user experience. Following a microservices pattern, the UI operates independently and integrates seamlessly with our data.

8.1 Secure Authentication (Login Page)

A simulated Google Sign-In page was integrated to demonstrate secure user entry workflows.

8.2 Product Catalog

The catalog dynamically renders items from the database, populated with high-quality product images.

8.3 Filter & Search Capabilities

Users can query products based on specific categories, price ranges, and string-matching.

8.4 Shopping Cart

React hooks manage state for the shopping cart, instantly reflecting price totals and quantities.

8.5 User Dashboard

Customers can view their account status and past orders.

8.6 Admin Dashboard

Administrators have full CRUD (Create, Read, Update, Delete) access to manage the inventory.

9. Conclusion

This project successfully demonstrates the end-to-end design of a scalable E-commerce Database Management System. By strictly enforcing relational integrity up to 3NF, utilizing advanced SQL triggers for automated data consistency, and decoupling the User Interface from the database logic, the architecture mirrors modern enterprise software standards.

Future Enhancements:

  • Migration of the mock frontend JSON API to a live Express.js REST API that directly interfaces with the MariaDB database.
  • Implementation of a live Stripe Payment Gateway integration for real-world checkout scenarios.
  • Deployment to a cloud environment (e.g., AWS or Render) for public access.
